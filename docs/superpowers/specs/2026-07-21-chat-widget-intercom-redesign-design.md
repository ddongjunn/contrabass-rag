# 챗봇 위젯 UI/UX 리디자인 — Intercom 스타일 + 다크모드 설계

> `chat-widget.js`/`chat-widget.css`(정적 자산, `ragbot-server` 서빙)의 시각/모션 전면 개편.
> 백엔드 API 계약(`/api/chat` 요청/응답)은 변경 없음 — 이 스펙은 순수 프론트엔드 범위.

## 1. 배경 / 문제

현재 챗봇 위젯은 다음 문제가 있다(사용자 피드백, 2026-07-21):

- 패널 열림/닫힘이 `display:none`↔`flex` 전환뿐이라 애니메이션이 없고 뚝뚝 끊겨 보인다.
- 패널 크기가 `388×640px`로 고정돼 있어 위젯(대시보드형 콘텐츠)이 많아지면 확대할 방법이 없다.
- 라이트 모드 전용이라 다크모드 요청에 대응 못 한다.
- 브랜드 오렌지가 채팅 셸용(`--brand` #f36c21)과 위젯용(`--primary` #f76205)으로 미묘하게 다르게
  두 벌 정의돼 있어 색이 통일돼 보이지 않는다.
- 아이콘이 CSS 도형(런처)·유니코드 문자("×")·이모지(🔍)로 제각각 그려져 있어 톤앤매너가 불일치한다.
- 위젯(`metric_rank`/`project_usage_bar`) 행의 이름·값 칸이 `min/max-width` 범위로만 잡혀 있어
  행마다 텍스트 길이에 따라 정렬선이 어긋난다.

## 2. 목표 / 비목표

**목표**
- Intercom(intercom.com) 챗봇의 시각적 톤앤매너(모션, 레이아웃 구조) 참고 — 다크모드 기본,
  라이트 토글 지원, 가벼운 인터랙션(안읽음 뱃지) 추가.
- 아이콘을 Google Material Symbols SVG로 통일(직접 드로잉 금지).
- 위젯 5종(`metric_rank`/`inventory_count`/`status_donut`/`threshold_banner`/`project_usage_bar`)의
  정렬·색상을 `dataviz` 스킬(검증된 팔레트·마크 스펙) 기준으로 정리.
- 사용자가 패널 크기를 드래그로 직접 조절 가능하게.
- 신규 외부 의존성 0개 — 지금처럼 바닐라 ESM, 빌드 스텝 없음 유지.

**비목표 (이번 스펙 밖)**
- **멀티턴 히스토리/대화 목록**(Intercom의 inbox 기능) — 루트 `CLAUDE.md`가 명시적으로 금지.
  이번 리디자인은 시각/모션 + 가벼운 인터랙션(안읽음 뱃지)까지만.
- 백엔드 API·위젯 JSON 계약 변경 없음.
- 사운드 알림, 푸시 알림 등 브라우저 알림 기능.
- `remote-contrabass-admin`(포털) 쪽 코드 변경 없음 — 위젯은 여전히 자체 완결형 스크립트.

## 3. 컬러 토큰 체계

역할을 3갈래로 분리한다(지금처럼 브랜드색을 위젯에도 재사용하지 않음):

| 역할 | 용도 | 값(라이트/다크 공용) |
|---|---|---|
| 브랜드/인터랙션 | 런처, 활성 상태, 유저 말풍선, 포커스 링 | `#f76205`(기존 유지 — 회사 브랜드) |
| 상태(severity) | 위젯 GOOD/WARN/CRIT, 에러 말풍선 | good `#0ca30c` · warn `#fab219` · crit `#d03b3b`(dataviz 검증 status palette, 테마 불변) |
| 중립 크기(accent) | severity 없는 수치바(예: 네트워크 B/s) | 라이트 `#2a78d6` / 다크 `#3987e5`(dataviz sequential blue) |

잉크/서페이스(다크 기본, 라이트는 `[data-theme="light"]`로 오버라이드):

| 역할 | 다크(기본) | 라이트 |
|---|---|---|
| 패널 배경 | `#1a1a19` | `#ffffff` |
| 메시지 영역 배경 | `#0d0d0d` | `#f9f9f7` |
| 기본 텍스트 | `#ffffff` | `#0b0b0b` |
| 보조 텍스트 | `#c3c2b7` | `#52514e` |
| 흐린 텍스트(축/라벨) | `#898781`(공용) | `#898781`(공용) |
| 구분선 | `#2c2c2a` | `#e1e0d9` |

에러 말풍선 색은 critical과 동일한 `#d03b3b`로 통일(별도 danger 색 폐기).

**타이포그래피**: `Inter` 유지(이미 트렌디하고 문제의 근원이 아니었음). 크기 스케일을
11/12/13/14/18px로 고정 — 위젯 텍스트가 제각각이던 걸 여기서 정리.

## 4. 아이콘 체계

Google Material Symbols(Apache License — SVG path를 코드에 직접 임베드 가능, CDN/폰트 파일 불필요)에서
아래 아이콘만 골라 `render/icons.js`에 SVG path 상수로 번들한다. `fill="currentColor"`로 둬서
테마별 잉크 색을 자동으로 물려받는다(별도 아이콘 색 토큰 불필요).

| 용도 | 아이콘 | 대체하는 것 |
|---|---|---|
| 런처 버튼 | `chat` | CSS로 그린 말풍선 도형 |
| 패널 닫기 | `close` | "×" 문자 |
| 라이트/다크 토글 | `light_mode` / `dark_mode` | (신규) |
| 전송 버튼 | `send` | "전송" 텍스트 |
| 어시스턴트 아바타 | `support_agent` | 글자 아바타 |
| 빈 상태 | `search_off` | 🔍 이모지 |
| 위젯 배너 상태 | `info` / `warning` / `error` | (신규 — 지금은 아이콘 없음) |
| 리사이즈 그립 | 대각선 3줄 커스텀 SVG(범용 리사이즈 그립 관례, Material Symbols에 대응 아이콘 없음) | (신규) |

## 5. 챗 셸 구조 + 애니메이션

**런처(Intercom 방식)**: 원형 FAB(56px), 패널이 열려도 사라지지 않고 같은 자리에서
아이콘만 `chat`↔`close`로 모핑(크로스페이드+회전, 150ms). 안읽음 뱃지(숫자 원, 9 초과 시 "9+")를
오른쪽 위에 표시, 패널 열면 사라짐.

**패널 애니메이션**: 열릴 때 `opacity 0→1` + `scale(0.92→1)` + `translateY(12px→0)`,
220ms `cubic-bezier(0.16,1,0.3,1)`(오른쪽 아래에서 펼쳐지는 느낌). 닫힐 때 역방향 160ms.
`prefers-reduced-motion`이면 즉시 전환(기존 미디어쿼리 유지).

**리사이즈**: 패널 왼쪽 위 모서리에 드래그 그립. 최소 340×440, 최대는 뷰포트 90%.
크기는 `localStorage`(`contrabass.chat.panelSize`)에 저장, 다음 로드 시 뷰포트보다 크면 clamp.
모바일 브레이크포인트(≤520px, 패널이 이미 거의 풀스크린)에서는 그립을 숨긴다.

**헤더**: kicker+제목 유지, 테마 토글 버튼 추가, 닫기 버튼 유지(모핑 런처와 별개로 명시적 닫기 수단
하나는 접근성상 남김).

**메시지**: 새 메시지 fade-in + `translateY(6px→0)` 180ms. 타이핑 인디케이터(점 3개 바운스)는
유지하되 다크 서페이스 대비로 색 재조정.

**입력창**: 전송 버튼을 아이콘(`send`) 버튼으로 교체. 자동 줄바꿈 textarea 구조는 유지.

## 6. 위젯 5종 정렬·비주얼 정리

**정렬 버그의 근본 원인**: `metric_rank`/`project_usage_bar`의 이름 칸이
`min-width:76px, max-width:132px`처럼 범위로 잡혀 있어 행마다 텍스트 길이에 따라 경계가 어긋난다.

**수정**: 이름 칸 고정폭 100px(초과 시 말줄임), 값 칸 고정폭 64px(오른쪽 정렬,
`tabular-nums` 유지 — 열 정렬용이라 dataviz 권장에 부합).

**막대 마크(dataviz 스펙 적용)**: 지금은 막대 양끝이 다 둥근데(`border-radius:6px` 전체),
원칙은 "기준선(왼쪽)은 각지고 값 끝(오른쪽)만 둥글게"(`border-radius: 0 4px 4px 0`)다.
트랙(배경 레일)은 지금처럼 완전히 둥근 알약 모양 유지(막대는 데이터 마크, 트랙은 UI 크롬이라
별도 취급). 막대 색은 3장 토큰 그대로: severity 있으면 상태색, 없으면 파랑 accent.

**status_donut**: 세그먼트 사이 2px 서페이스 갭 추가(dataviz "stacked segment gap" 규칙) —
지금은 세그먼트가 붙어 있어 하나로 뭉쳐 보일 수 있다.

**inventory_count**: 구조(큰 숫자 stat tile) 유지, 색만 새 토큰. 숫자는 proportional
figures 유지(이미 tabular-nums 안 씀 — dataviz 권장과 일치, 변경 불필요).

**threshold_banner**: 배너 안에 `info`/`warning`/`error` 아이콘 추가(지금은 색 배경만 있고
아이콘 없음).

## 7. 상태 저장

| 키 | 저장 위치 | 기본값 | 비고 |
|---|---|---|---|
| `contrabass.chat.theme` | `localStorage` | `dark` | shadow host에 `data-theme` 속성으로 적용 |
| `contrabass.chat.panelSize` | `localStorage` | 없음(CSS 기본값 사용) | `{width, height}`, 로드 시 뷰포트로 clamp |
| 안읽음 카운트 | 메모리(세션 한정) | 0 | 새로고침 시 리셋 — 영구 저장 불필요 |

## 8. 파일 구조

**신규**:
- `render/icons.js` — Material 아이콘 SVG path 상수 + `icon(name, size)` 헬퍼
- `render/theme.js` — 테마 읽기/쓰기/적용(`getTheme`/`setTheme`/`applyTheme`)

**수정**:
- `chat-widget.css` — 토큰 전면 개편(다크 기본 + `[data-theme="light"]` 오버라이드), FAB 런처,
  리사이즈 그립, 애니메이션, 위젯 고정폭 칸·막대 radius·도넛 갭
- `render/chrome.js` — 런처(아이콘 기반)·헤더(테마 토글)·리사이즈 그립 마크업
- `chat-widget.js` — 테마 토글 클릭 핸들러, 리사이즈 포인터 드래그, 안읽음 카운터, 패널 크기 저장/복원 배선
- `render/widgets/metric-rank.js`, `project-usage-bar.js`, `status-donut.js`, `threshold-banner.js`,
  `inventory-count.js` — 고정폭 클래스·아이콘 적용

**신규 의존성 없음** — 순수 ESM, 빌드 스텝 없음 유지.

## 9. 에러 처리

- fetch 실패 시 현재처럼 `message[data-error="true"]` 말풍선 렌더 — 색만 critical 상태색으로 통일(§3).
- 리사이즈 저장값이 손상되거나(JSON 파싱 실패) 현재 뷰포트보다 크면 CSS 기본값으로 폴백.
- 알 수 없는 테마 값이 `localStorage`에 있으면 `dark`로 폴백.

## 10. 테스트 방법

기존 `node --test` 하네스 확장.

- `icons.js`: 알려진 아이콘 이름마다 유효한 SVG 마크업을 반환하는지, 모르는 이름은 무엇을 반환하는지.
- `theme.js`: 저장/조회 순수 함수 테스트, 손상된 값 폴백.
- 리사이즈: 크기 clamp 계산을 순수 함수로 분리(`clampPanelSize(w, h, viewport)`)해서 테스트.
  실제 포인터 드래그 배선(DOM/포인터 이벤트)은 지금 마운트 로직과 동일하게 브라우저 수동 확인으로 남긴다.
- 위젯 렌더러: 기존 XSS 회귀 테스트 유지 + 신규 고정폭 클래스·아이콘 마크업 존재 여부 검증 추가.
- `dataviz` 스킬의 `scripts/validate_palette.js`로 최종 컬러 세트(라이트/다크 각각)를 검증 —
  특히 브랜드 오렌지가 텍스트/배지 위 텍스트와 충분한 대비를 내는지 별도 확인 필요(§11 열린 리스크).

## 11. 열린 리스크 (구현 전 확인 필요)

- Material Symbols SVG path는 이 스펙에 아이콘 **이름**만 정의했다 — 실제 path 데이터는 구현
  단계에서 공식 저장소(Apache 2.0)에서 가져와야 한다.
- 브랜드 오렌지(`#f76205`)는 dataviz 검증 팔레트에 없는 커스텀 색이라, 이 색 위에 흰 텍스트/뱃지
  숫자를 얹었을 때 다크·라이트 양쪽에서 대비 기준(4.5:1 텍스트 기준)을 통과하는지 구현 후
  `validate_palette.js`로 별도 확인해야 한다. 실패하면 브랜드색은 유지하되 텍스트 색만 조정.
- 리사이즈 드래그는 마우스 기준으로 설계했다 — 터치 디바이스(포인터 이벤트로 구현 시 자동 커버되긴
  하나) 실기기 확인 필요.
- 안읽음 뱃지의 정확한 트리거 조건(예: 패널이 백그라운드 탭에 있을 때도 카운트할지)은 구현 중
  브라우저 `visibilitychange` API 사용 여부를 포함해 세부 조정 여지가 있다.

## 12. 검토했던 다른 방식 (선택 안 함)

- **Google Fonts CDN으로 Material Symbols 로드**: 구현이 한 줄로 가장 간단하지만, 사내 포털
  임베드라는 배포 맥락상 외부망 차단 시 아이콘이 통째로 깨질 위험이 있어 기각. 인라인 SVG 번들 채택.
- **CSS 네이티브 `resize: both`**: 구현이 훨씬 간단하지만 브라우저 기본 그립 모양을 스타일링할
  수 없어(못생긴 삼각형 고정) 트렌디한 톤앤매너 요구와 안 맞음. 커스텀 JS 드래그 핸들 채택.
- **확장(maximize) 토글 버튼 방식**: Intercom의 실제 기본 동작에 더 가깝지만, 사용자가 명시적으로
  드래그 리사이즈를 선택함.
- **라이트 기본 + 다크 토글 / 포털 테마 자동 연동**: 사용자가 다크를 기본으로 원함 — 라이트는
  토글로만 제공.
- **Intercom처럼 대화 히스토리/목록 포함**: 루트 `CLAUDE.md`의 "웹 위젯 멀티턴 히스토리 지원 금지"
  불변식과 정면으로 충돌해 기각. 시각/모션 + 가벼운 인터랙션까지만 반영.

## 13. 후속 (이번 스펙 밖)

- 위젯에 새로운 종류(예: 시계열 라인 차트)가 추가되면 이 스펙의 §3·§6 팔레트/마크 규칙을 그대로
  확장 적용한다(새로 정의할 필요 없음).
- 안읽음 뱃지 고도화(사운드, 브라우저 알림)는 별도 스펙으로 분리.
